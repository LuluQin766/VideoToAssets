## **阶段0：视频元信息抓取（新增，必须前置）**

  

这一阶段要放在最前面，因为你后面做公众号文章、摘要、引用来源、高光标注，都需要这个信息。

  

### **目标**

### **建议抓取内容**

- 视频标题

- 视频 ID

- 字幕可用情况
    
- 自动字幕 / 人工字幕类型

### **输出建议**

```
/output/{video_id}/metadata/
├── video_info.json
├── video_info.md

```

### **作用**

- 内容来源是谁

- 视频热度如何

这个阶段是你现在流程里**最缺的一环**。

## **阶段1：字幕优先下载**

  

你原来的流程可以保留，但建议补几个细节。

  

### **流程**

1. 检查字幕列表
    
2. 优先下载人工字幕
    
3. 人工字幕没有，再下载自动字幕
    
4. 多语言同时下载
    
5. 验证字幕完整性
    

  

### **建议补充**

- 优先级建议：
    
    **人工字幕 > 官方字幕 > 自动字幕 > Whisper 转写**
    
- 下载后要做字幕质量检查：
    
    - 字幕文件是否为空
        
    - 总行数是否过少
        
    - 是否只有几句重复内容
        
    - 是否存在大量乱码
        
    - 时间轴是否连续
        
    

  

### **输出**

```
/output/{video_id}/subtitles_raw/
├── original.en.srt
├── original.zh-Hans.srt
├── auto.en.vtt
└── auto.zh-Hans.vtt
```

## **阶段2：Whisper / ASR 兜底转写**

  

只有阶段1失败，或者字幕质量过低时才进入。

  

### **建议补充判断条件**

  

除了“下载失败”以外，也要在以下情况下触发 ASR：

- 字幕只有很少几行
    
- 字幕中断明显
    
- 自动字幕错得很离谱
    
- 语言识别不正确
    
- 时间轴缺失或异常
    

  

### **建议输出**

```
/output/{video_id}/asr/
├── audio.m4a
├── whisper_raw.srt
├── whisper_raw.json
└── asr_log.txt
```

### **说明**

  

如果以后要做高光切片、语义片段聚类、情绪波峰检测，最好保留：

- 逐句时间戳
    
- segment-level JSON
    
- confidence 分数（如果能拿到）

## **阶段3：字幕标准化与清洗**

  

这个阶段建议单独拆出来，不要直接“转换输出”，而是先做**标准化**。

### **处理内容**

- 统一编码为 UTF-8
    
- 统一时间戳格式
    
- 去掉 HTML 标签、转义符、噪声字符
    
- 合并重复字幕
    
- 纠正明显断句问题
    
- 统一标点
    
- 区分“原始文本”和“清洗文本”
    
- 对中英文字幕做对齐（如果有双语）
    

  

### **输出**

```

└── cleaned_plain.txt
```

### **阶段2：Whisper / ASR 语音识别兜底**

- 下载音频
    
- 使用本地 ASR 模型进行转写
    
- 输出带时间戳的 .srt / .json 文件

- 统一编码与格式

- 输出清洗后的标准字幕、纯文本、时间戳文本和 Markdown 版本
    

  

### **阶段4：格式转换与基础输出**

- 建立标准目录结构
    
- 将字幕转换为：
    
    - 原始字幕文件
        
    - 带时间戳文本
        
    - 纯文本稿
        
    - Markdown 版本
        
    - 双语版本（如可用）
        
    

  

### **阶段5：大模型质检与重整**

- 检查字幕和文本是否完整、是否有漏句、重复、错词、语义断裂等问题
    
- 生成质量报告和修复建议
    
- 基于清洗稿生成一版由大模型整理后的正式文稿，包括：
    
    - 可阅读全文版
        
    - 主题重组版
        
    - 去口语化版
        
    - 保留原说话风格版
        
    

  

### **阶段6：视频内容提炼与知识整理**

- 生成一句话摘要、标准摘要、主题提纲、结构导图式内容
    
- 提取金句、核心观点、行动建议
    
- 输出适合阅读和后续内容创作的整理稿

#### **7.1 公众号长文**

- 基于完整文案和提炼稿生成 3 篇公众号文章
    
- 每篇包含：选题、标题、导语、分节排版、正文、结尾互动、来源说明

- 不伪装为原创观点，而是定位为资料整理、学习笔记、内容提炼
    

  

#### **7.2 小红书短文**

- 生成 5 篇适合小红书发布的短文
    
- 每篇采用轻量化、强钩子、强观点、强共鸣结构
    
- 配套标题、正文、结尾互动句、标签建议
    

  

### **阶段8：高光片段识别与切片清单生成**

- 从字幕或完整文案中挖掘高价值传播片段
    
- 对于约 1 小时视频，至少产出 30 个高光片段候选
    
- 采用分批策略：粗筛 → 精选 → Top10
    
- 每个片段记录：
    
    - 起止时间
        
    - 标题
        
    - 完整文案
        
    - 截取原因
        
    - 高光类型
        
    - 传播价值说明
        
    

  

### **阶段9：来源归档与最终索引**

## **Prompt 1：大模型质检 Prompt**

```

3. 是否存在明显的 ASR 误识别

6. 是否还需要进一步清洗和重整

输出要求：

B. 列出发现的问题，尽量具体

```

## **Prompt 2：大模型整理全文 Prompt**

```
你现在是一个“视频文稿整理编辑助手”。

我会提供一份已经过初步清洗的视频字幕/转写文本。请你在不篡改原意的前提下，将它整理成一份更适合阅读和后续内容创作的正式文稿。

整理要求：
1. 保留原始信息，不虚构、不补造事实
2. 修正口语化造成的断裂和重复
3. 合理分段，增强可读性
4. 必要时补足主语，使句子更完整
5. 删除明显的语气词、口头重复、无意义停顿
6. 保持原视频的核心观点、说话顺序和论述逻辑
7. 如果原文中提到人名、节目名、书名、机构名，请尽量保留
8. 不要写成总结，不要丢失细节，要尽量接近“整理后的完整文稿”

输出格式：
- 标题：整理后的完整文稿
- 正文：按自然段排版
- 最后附上一个“整理说明”，说明你主要做了哪些整理

```

# **五、最后给你的一个总建议**

  

你现在这个项目，本质上已经不是“下载字幕工具”了，

而是在往一个 **视频内容知识加工流水线** 的方向走。

  

所以你后面设计时，最好不要把重点只放在：

- 能不能下载字幕
    
- 能不能转成 txt
    

  

而是要把重点放到：

- **内容是否可追溯**
    
- **整理后是否可读**
    
- **是否可批量生产内容**
    
- **是否支持高光切片**
    
- **是否支持后续知识库沉淀**
    
- **是否支持不同平台二次创作**

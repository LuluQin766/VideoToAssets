
# **给 Codex 的完整首轮开发任务书**


## **0. 项目背景**

  

项目名：VideoToAssets

  

目标：

构建一个 Python 工程，把单个视频 URL 加工为一套结构化内容资产，包括：

- 视频元信息抓取
    
- 字幕优先下载
    
- ASR 兜底转写
    
- 字幕清洗与标准化
    
- 多格式文本导出
    
- 大模型质检
    
- 大模型整理后的完整文稿
    
- 为后续摘要、公众号文章、小红书短文、高光切片打基础
    

  

当前已有一个现成目录：ytdlp_v2

本轮要求先将其作为外部模块保留到：

```
external/ytdlp_v2/
```

不要一开始大拆旧代码。优先做**可运行的首轮闭环**。

---

## **1. 本轮开发目标**

  

本轮只实现 **MVP v1**，范围如下：

  

### **必做功能**

1. 接收单个视频 URL
    
2. 抓取视频 metadata
    
3. 尝试下载字幕
    
4. 如果字幕不存在或质量差，回退到 Whisper / ASR
    
5. 清洗字幕与转写文本
    
6. 导出标准文本文件
    
7. 调用 LLM 做质检
    
8. 调用 LLM 生成整理后的可阅读完整文稿
    
9. 为该视频生成标准输出目录与 README.md
    

  

### **暂不做**

- 批量任务调度
    
- 公众号 3 篇文章生成
    
- 小红书 5 篇短文生成
    
- 高光片段自动筛选
    
- 多平台适配
    
- Web UI
    

  

这些内容本轮只预留接口，不要求做完。

### **模块 D：字幕下载**

  

#### **任务**

实现字幕优先下载逻辑：

1. 检查可用字幕
    
2. 优先下载人工字幕
    
3. 其次下载自动字幕
    
4. 支持语言优先级配置，例如：
    
    - zh-Hans
        
    - zh
        
    - en
        
    

  

建议文件：

```
src/video_to_assets/collectors/subtitle_fetcher.py
src/video_to_assets/validators/subtitle_validator.py
```

#### **输出文件**

```
subtitles_raw/
```

例如：

```
original.zh-Hans.srt
original.en.srt
auto.zh.vtt
```

#### **质量检查规则**

至少实现这些检查：

- 文件存在
    
- 文件非空
    
- 字幕条目数大于最低阈值
    
- 不是只有几句重复内容
    
- 时间戳格式基本正常
    

  

#### **验收标准**

- 成功下载字幕时，不进入 ASR
    
- 字幕质量太差时，能触发 ASR fallback
    
- 下载状态和验证结果有日志输出

建议文件：

#### **验收标准**

- 无字幕视频可成功走 ASR
    
- 至少输出一份带时间戳文本
    
- ASR 失败时有明确报错信息
    
- 不影响 metadata 阶段产物保留
